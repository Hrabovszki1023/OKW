14 ### Reines OKW Testprojekt anlegen
15 
16 #### Ein neues Maven Projekt anlegen.
17 
18 ```java
19  StartApp("Eclipse");
20  SelectWindow("Eclipse")
21  SelectMenu("File->New->Project...");
22 
23  SelectWindow("New Project");
24  Select( "Project Type", "Maven/Maven Project");
25  ClickOn( "Next");
26 
27  Select("Create a Simple Project", "CHECKED");
28  Select("Use default" Workspace set, "CHECKED");
29  ClickOn( "Next");
30 
31  SetValue("Group Id", "de.openkeyword");
32  SetValue("Artifact Id", "OKW_Tutorial");
33  SetValue("Version", "${IGNORE}") // => den "*Snapshot"-wert übernehmen
34 
35  // optional
36  SetValue("Name", "Add here Artifact Name");
37  SetValue("Description", "Add here your Artifact Description");
38 
39  ClickOn( "Finish");
40 ```
41 
42 #### Entstandene pom.xml-Datei anpassen.
43 
44 Dazu im `Project Explorer` die `pom.xml` öffnen:
45 ```java
46  DoubleClickOn("Project Explorer", "OKW_Tutorial/pom.xml)
47 ```
48 
49 ##### 1. Java 8 einstellen
50 Die Java Version im ```maven-compiler-plugin``` Prüfen und ggf. einstellen. - Wir benötigen Java 1.8.
51 Im pom-Editor unten den Reiter `pom.xml` auswählen. Wenn das Projekt frisch angelegt wurde, dann sollte der folgende Abschnitt nach `project/description>` eingefügt werden. (einfach kopieren und unter `<description>` einfügen)
52 ```xml
53 <build>
54  <plugins>
55  <plugin>
56  <artifactId>maven-compiler-plugin</artifactId>
57  <version>3.2</version>
58  <configuration>
59  <source>1.8</source>
60  <target>1.8</target>
61  </configuration>
62  </plugin>
63  </plugins>
64 </build>
65 ```
66 
67 ##### 2. OKW dependency eingeben
68 Nun auf den Reiter `Dependencies` gehen und dort
69 
70 ```java
71 SelectWindow("Dependency Properties")
72 SetValue( "Group Id", "de.openkeyword" )
73 SetValue( "Artifact Id", "se" )
74 SetValue( "Version", "0.0.23" )
75 
76 // Alle anderen Werte werden nicht verändert.
77 // SetValue( "Classifier", "${IGNORE}" )
78 // SetValue( "Type", "${IGNORE}" )
79 // SetValue( "Scope", "${IGNORE}" )
80 // SetValue( "System Path", "${IGNORE}" )
81 // SetValue( "Optional", "${IGNORE}" )
82 ```
83 > **Anmerkung 1:** `Version` ggf. anpassen.
84 
85 > **Anmerkung 2:** Um OKW mit HTML/Selenium zu verwenden muss lediglich `se` eingebunden werden.
86 D.h. OKW `core` wird automatisch durch Maven miteingebunden.
87 
88 Alternativ kann der Folgende block im Reiter `pom.xml` unter `dependencies` eingefügt werden:
89 ```xml
90  <dependency>
91  <groupId>de.openkeyword</groupId>
92  <artifactId>se</artifactId>
93  <version>0.0.23</version>
94  </dependency>
95 ```
96 
97 eingeben.
98 
99 ##### 3. JUnit
100 JUnit wird wie OKW `se` eingebunden: Dazu müssen folgende Werte auf dem Tab Dependencies im pom-Editor eingeben:
101 Wie bei OKW ggf. die Versions Nummer aktualisieren:
102 
103 ```xml
104  <dependency>
105  <groupId>junit</groupId>
106  <artifactId>junit</artifactId>
107  <version>4.12</version>
108  <scope>test</scope>
109  </dependency>
110 ```
