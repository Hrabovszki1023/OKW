3 ## Einen Testfall mit OKW in JUnit erstellen: "Hello Test" - Teil 2
4 
5 ### 1. JUnit TestKlasse anlegen.
6 
7 Eine JUnit Klasse wird folgender maßen angelegt:
8 
9 ```java
10 EN.SelectWindow("EcLipse");
11 EN.SelectMenu("File->New->JUnit Test Case");
12 
13 EN.SelectWindow("New JUnit Test Case");
14 EN.SetValue("New JUnit 4 test")
15 EN.SetValue("Source Folder", "OKW_Tutorial/src/test/java")
16 
17 EN.SetValue("Package", "${IGNORE}");
18 EN.SetValue("Name", "Hello_OKW");
19 EN.SetValue("Superclass", "${IGNORE}");
20 
21 EN.SetValue("setUpBeforeClass", "UNCHEKED");
22 EN.SetValue("setUp", "UNCHEKED");
23 EN.SetValue("tearDownAfterClass", "UNCHEKED");
24 EN.SetValue("tearAfter", "UNCHEKED");
25 ```
26 
27 ### 2. Eine Testcase-Methode einfügen.
28 
29 Nachdem Anlegen sollte die erzeugte Klasse bzw. die Datei in Eclipse geöffnet und angezeigt werden.
30 
31 Als Testfall legen wir `tcHello_OKW()` an.
32 
33 ```java
34  @Test
35  public void tcHello_OKW() throws Exception
36  {
37  EN.BeginTest( "Hello Test" );
38  // Add test steps here...
39  EN.EndTest();
40  }
41 ```
42 
43 Die beiden Schlüsselwörter bedeuten:
44 
45 * `EN.BeginTest( "Hello Test" );` - Initialisiert OKW: Wenn mehrere Testfälle hintereinander ausgeführt werden und einer der Testfälle schlägt fehl (z.B. wegen einer Abweichung oder Exception), dann wird mit BeginTest OKW wieder in den Initialzustand versetzt. Technisch wechselt ggf. OKW vom Zustand NOK nach OK.
46 
47 * `EN.EndTest();` - Löst Aufräum-Aktionen aus, sofern diese definiert worden sind und es wird dem Logging mit geteilt das nun der Testfall beendet ist.
48 
49 > Das Verschachteln von BeginTest/EndTest ist nicht erlaubt.
50 
51 
52 Nun wird der java-compiler `EN` anmeckern, weil wir OKW noch nicht der Klasse "vorgestellt" haben:
53 Dazu vor oder nach (wie´s gefällt) `import org.junit.Test;` folgende Zeile einfügen:
54 
55 ```java
56 import okw.core.EN;
57 ```
58 
59 Alles zusammen sollte etwa so aussehen:
60 
61 ```java
62 import org.junit.Test;
63 
64 import okw.core.EN;
65 
66 public class Hello_OKW{
67 
68  @Test
69  public void tcHello_OKW() throws Exception
70  {
71  EN.BeginTest( "Hello Test" );
72  // Add test steps here...
73  EN.EndTest();
74  }
75 }
76 ```
77 
78 Damit ist schon ein minimal JUnit-Rahmen für die Erstellung und Ausführung von Testfällen mit OKW fertig.
79 
80 ### 3. Testfall `tcHello_OKW` ausführen
81 
82 ```java
83 EN.SelectWindow("EcLipse");
84 EN.SelectContext("Project Explorer", "/OKW_Tutorial/src/test/java/(default package)/Hello_OKW.java", );
85 EN.SelectMenu("Run->Run As->JUnit Test");
86 ```
87 
88 ## Result
89 Das Result-Log von OKW ist etwas länger, so wird u.a. eine Liste der gefundenen Frames ausgegeben, aber die ent scheidende Ausgabe ist am Ende zu finden:
90 
91 ```
92 -------------------------------------------------------------------------
93 Testcase: "Hello Test"
94  Memorize value of MemKey 'TCN'...
95  Value('TCN') = 'Hello Test'
96 -------------------------------------------------------------------------
97 ```
98 
99 > Die beiden Zeilen unter `Testcase: "Hello Test"` zeigen an, das OKW sich den Testfall-Namen gemerkt hat.
100 mit `${TCN}` kann dieser in Schlüsselwörtern verwendet werden.
101 
102 Damit haben wir den ersten OKW Testfall erstellt.
103 
104 Im nächsten Tutorial werden wir einen einfachen aber etwas längeren Testfall in OKW Schlüsselwort-Notation erstellen.
