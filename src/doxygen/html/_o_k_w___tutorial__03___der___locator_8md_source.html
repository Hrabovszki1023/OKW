1 # OKW Tutorial 3: Der Locator
2 
3 ## Der Locator
4 Des Locator eines GUI-Objektes enthält die Information **_wie_** ein Objekt erkannt wird.
5 Der Locator für die OKW Se-Klassen wird als _XPATH_ angegeben.
6 
7 ## Ziele des Locators
8 
9 1. Ein GUI-Objekt muss eindeutig erkannt und wiedererkannt werden können.
10 2. Der Locator eines GUI-Objektes wird genau an einer Stelle in einem Frame definiert und gepflegt. - **DRY-Prinzip**
11 3. Der Locator muss für jedes GUI-Objekt exakt angepaßt werden können.
12 
13 ## Die Klasse 'OKWLocator'
14 
15 In OKW existiert eine eigenen Klasse, die Klasse `okw.gui.OKWLocator` für die Definition von Locatoren.
16 Die Locatoren können hierarchisch über GUI-Adapter hinweg miteinander verkettet werden. Dabei definieren die GUI-Adapter jeweils nur ihren Teil eines Loacators.
17 
18 >Eine eigene Klasse für die Locatoren ist notwendig, da die Erkennungseigenschaften von GUI-Objekten
19 meist nicht mit einfach zudefinierenden `id`'s festgelegt werden kann. - In vielen Projekten wird die Problematik der Objekterkennung für den Test nicht hinreichen gewürdigt und es werden sehr komplexe Locatoren notwendig.
20 
21 **Wie das funktioniert ist Gegenstand dieses und der der folgenden Tutorials.**
22 
23 > Die 'okw.gui.OKWLocator' Klasse ist für die Verkettung von 'String'-Werten entworfen worden.
24 D.h. sie **nicht** ausschliesslich nur für _XPATH_ geeignet, sonder sie kann für _beliebig andere Locatoren, die auf String-Werten basieren_, verwendet werden.
25 
26 ## Definition eines _einfachen_ Locators
27 Der Locator für ein GUI-Objekt wird jeweils im Frame (für Fortgeschrittene: und im GUI-Container) eines GUI-Objektes definiert.
28 
29 Wir fangen mit dem einfachsten Fall an. Als Beispiel verwenden wir das Login-Fenster von Word-Press.
30 In diesem speziellen Fall haben alle Elemente jeweils eine Eindeutige `id` als Attribute.
31 
32 D.h. wir finden _jedes_ GUI-Object im WP-Login nach dem Prinzip: "Suche das Element mit der 'id', die einen bestimmten Wert" hat.
33 
34 >**Hinweis:** Wichtig:** Dieser Wert sollte über alle Releases des Produkte const. sein
35 
36 ###Konkretes Beispiel `Username`
37 
38 Der Locator wird im Konstruktor eines GUI-Adapters als Parameter wie folgt festgelegt:
39 
40 **Beispiel:**
41 ```java
42 @OKW (FN="Username") // FN des GUI-Objektes
43 public SeInputText Username = new SeInputText( "//*[@id='user_login']" );
44 ```
45 
46 Entscheidend ist in diesem Beispiel der Ausdruck `"//*[@id='user_login']"`.
47 
48 Das ist der einfachste Fall für die definition eines Locators. Das Eingabefeld 'User' wird anhand des Attributes 'id' eindeutig erkannt.
49 
50 Hinweis: Die _XPATH_ Angaben erläutern wir hier nicht, weil das für die OKWLocator-Klasse nicht relevant ist. Wichtig ist lediglich, dass das Feld _User_ mit der _XPATH_-Angaben eindeutig erkannt wird.
51 
52 Das ganze Frame für _WP-Login_ (Wordpress-Login) sieht wie folg aus:
53 
54 ```java
55 package okw.gui.frames;
56 
57 import okw.OKW;
58 import okw.gui.adapter.selenium.*;
59 
60 @OKW(FN="WP-Login")
61 public class wp_login extends SeBrowserChild
62 {
63 
64  @OKW (FN="Username")
65  public SeInputText Username = new SeInputText("//*[@id='user_login']" );
66 
67  @OKW (FN="Password")
68  public SeInputText Password = new SeInputText("//*[@id='user_pass']" );
69 
70  @OKW (FN="Remember Me")
71  public SeInputCheckbox Remember_Me = new SeInputCheckbox("//*[@id='rememberme']" );
72 
73  @OKW( FN = "Log In")
74  public SeInputButton Log_In = new SeInputButton( "//*[@id='wp-submit']" );
75 
76  @OKW( FN = "Lost your password")
77  public SeLink LostPasswd = new SeLink( "//p[@id='nav']/a" );
78 
79  @OKW( FN = "Back")
80  public SeLink Back = new SeLink( "//p[@id='backoblog']/a" );
81 
82  @OKW( FN = "Login Error")
83  public SeAnyWin LogError = new SeAnyWin( "//div[@id='login_error']" );
84 
85  public wp_login()
86  {
87  super("//body/div[@id='login']/..");
88  }
89 }
90 ```
