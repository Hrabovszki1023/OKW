56 * \brief
57 * OKW.OKW_Ini ist die Klasse zur Konfigurationsdatei OKW_Ini.xml.<br/>
58 *
59 * Die Klasse OKW_Ini hält alle Konfigurationsparameter vor und stellt
60 * diese als Eigenschaften(Properties) zur Verfügung.
61 *
62 * Die Eigenschaften werden in zwei Bereiche eingeteilt
63 *
64 * Name | Beschreibung
65 * ------------------------------ | -------------
66 * OKW.OKW_Ini.OKW_Enviroment | Umgebungseigenschaften, wie z.B. Pfade.<br/> Genaue Beschreibung in der referenzierten Klasse.
67 * OKW.OKW_Ini.OKW_CustomSettings | Allgemeine Einstellungen, wie TimeOuts <br/>Genaue Beschreibung in der referenzierten Klasse.
68 *
69 * Die Klasse ist als Singelton konstruiert.<br/>
70 * __Die Verwendung dieser Klasse erfolgt__ _nicht_ über den Konstruktor, sondern __ausschließlich
71 * über die Eigenschaft OKW.OKW_Ini.Instance.__<br/>
72 *
73 * ## Verwendung der Klasse OKW.OKW_Ini ##
74 * ### Paket einbinden ###
75 * Das Paket OKW enthält die Klasse OKW_Ini. Die kann mit
76 * ~~~~~~~~~~~~~{.py}
77 * use OKW;
78 * ~~~~~~~~~~~~~
79 * erfolgen.
80 *
81 * ### Instanz des Singelton holen ###
82 * Eine Instanz der Klasse wird wie folgt geholt:<br/>
83 * ~~~~~~~~~~~~~{.py}
84 * OKW_Ini myOKW_Ini = OKW_Ini.Instance;
85 * ~~~~~~~~~~~~~
86 *
87 * ### Wert einer Eigenschaft referenzieren ###
88 * ~~~~~~~~~~~~~{.py}
89 * String MeinWert myOKW_Ini.CustomSettings.TimeOutExists;
90 * ~~~~~~~~~~~~~
91 *
92 * ## Hinweis für OKW Entwickler ##
93 * ### Erweiterung der Propertys ###
94 * Im folgenden soll anhand eines Beispiels gezeigt werden, wie eine neue Eigenschaft
95 * _NewProperty_ der OKW_Ini hinzugefügt werden kann.
96 *
97 * Die neue Eigenschaft _NewProperty_ wird mit _"Default Value"_ initialisiert.
98 * Für _NewProperty_ wird wie folgt definiert:
99 * ~~~~~~~~~~~~~{.py}
100 * private String __NewProperty = "Default Value";
101 *
102 * [XmlElement("NewProperty")]
103 * public String NewProperty {
104 * get{ return __NewProperty; }
105 * set{ __NewProperty = value; }
106 * }
107 * ~~~~~~~~~~~~~
108 *
109 * __Anmerkung:__ Die _private_ Variable <code>__NewProperty</code> ist notwendig,
110 * weil die Eigenschaft <code>NewProperty</code>
111 * zunächst auf einen Default-Wert initialisiert wird. Da OKW_Ini automatisch eine
112 * vollständige OKW_Ini.xml-Datei erstellen soll, ist es nicht möglich via Attributdefinition
113 * die Eigenschaft auf einen default Wert zu setzen,
114 * da in <code>System.Xml.Serialization</code> nur Eigenschaften serialisert werden, die sich
115 * vom Default-Wert unterscheiden.
116 *
117 * ### Automatische Serialisierung ###
118 * Die so eingefügte neue Eigenschaft _NewProperty_ wird automatisch mit den anderen Eigenschaften
119 * in die _OKW_Ini.xml_ serialisiert/deserialisiert.
120 *
121 * ### Quellen:
122 * http://www.mkyong.com/java/jaxb-hello-world-example/
123 * \~english
124 * \brief
125 * OKW.OKW_Ini is the related class to the configuration file OKW_Ini.xml.<br/>
126 *
127 * The class OKW_Ini is holding in store all configuration parameters and is providing
128 * this properties.
129 *
130 * The properties are divided in two sections:
131 *
132 * Name | Description
133 * ------------------------------ | -------------
134 * OKW.OKW_Ini.OKW_Enviroment | Environmental characteristics, such as paths.<br/> Detailed description in the referenced class.
135 * OKW.OKW_Ini.OKW_CustomSettings | General settings such as time-outs <br/>Detailed description in the referenced class.
136 *
137 * The class is created as singleton.<br/>
138 * __The use of this class is__ _not_ done with the constructor, but __exclusively
139 * with the property OKW.OKW_Ini.Instance.__<br/>
140 *
141 * ## Usage of the class OKW.OKW_Ini ##
142 * ### Integration of package ###
143 * The package OKW contains the class OKW_Ini. The class can be affected with
144 * ~~~~~~~~~~~~~{.py}
145 * use OKW;
146 * ~~~~~~~~~~~~~
147 *
148 * ### Getting the instance of the singleton ###
149 * An instance of the class is taken as follows:<br/>
150 * ~~~~~~~~~~~~~{.py}
151 * OKW_Ini myOKW_Ini = OKW_Ini.Instance;
152 * ~~~~~~~~~~~~~
153 *
154 * ### Referencing the value of a property ###
155 * ~~~~~~~~~~~~~{.py}
156 * String MeinWert myOKW_Ini.CustomSettings.TimeOutExists;
157 * ~~~~~~~~~~~~~
158 *
159 * ## Hint for OKW Developers ##
160 * ### Extension of Properties ###
161 * The following should be shown by way of example, as a new property
162 * _NewProperty_ of OKW_Ini can be added.
163 *
164 * The new property _NewProperty_ is initialized with _"Default Value"_ .
165 * For _NewProperty_ will be defined as follows:
166 * ~~~~~~~~~~~~~{.py}
167 * private String __NewProperty = "Default Value";
168 *
169 * [XmlElement("NewProperty")]
170 * public String NewProperty {
171 * get{ return __NewProperty; }
172 * set{ __NewProperty = value; }
173 * }
174 * ~~~~~~~~~~~~~
175 *
176 * __Note:__ The _private_ variable <code>__NewProperty</code> is necessary,
177 * because the property <code>NewProperty</code>
178 * is initialized first with a Default-Value.
179 * Since OKW_Ini will automatically create a complete OKW_Ini.xml file,
180 * it is not possible to set the property via attribute definition to a default value,
181 * since <code>System.Xml.Serialization</code>, only properties serializing, which
182 * differ from the default value.
183 *
184 * ### Automatical Serializing ###
185 * The new property _NewProperty_ inserted this wax is automatically serialized/ deserialized
186 * with the other properties in the _OKW_Ini.xml_ .
187 *
188 *
189 * \~
190 * \author Zoltán Hrabovszki
191 * \date 2014-10-25/jn
